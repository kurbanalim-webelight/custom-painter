üìö Part 1: What is Canvas?

Definition

Canvas is a class in Flutter (from dart:ui) that represents a 2D drawing surface. It's the interface through which you give drawing commands to Flutter's graphics engine.

Think of Canvas as:

  • A digital drawing board
  • Your direct connection to the screen's pixels
  • A command interface for the graphics renderer
  • The "pen and paper" of Flutter custom graphics

Where Canvas Comes From

Canvas is provided automatically by Flutter when it calls your paint() method:

@override
void paint(Canvas canvas, Size size) {
  // 'canvas' is given to you by Flutter
  // You don't create it yourself
}

The Process:

  1. Flutter's rendering pipeline needs to draw your CustomPaint widget
  2. It creates a Canvas object connected to that screen region
  3. It calls your paint() method and passes the Canvas to you
  4. You issue drawing commands to the Canvas
  5. Canvas forwards commands to the GPU/rendering engine
  6. Your graphics appear on screen

Coordinate System

  • Origin (0, 0) is at the top-left corner
  • X increases going right ‚Üí
  • Y increases going down ‚Üì
  • Units are logical pixels (Flutter handles device pixel density)

üß∞ Part 2: Canvas Methods (A to Z)

Here's a detailed reference of commonly used Canvas methods. Each method receives the current Canvas state (transformations, clip, etc.) and draws into the active painting region of your CustomPaint.

clipPath

void clipPath(Path path, { bool doAntiAlias = true })

Purpose: Restricts subsequent drawing operations to only appear within the given Path shape.

Parameters:

  • path ‚Äì The Path defining the clipping region.
  • doAntiAlias ‚Äì Whether to smooth the edges (default: true).

What it does:

  • Creates a mask from the path.
  • Any drawing outside this path becomes invisible.
  • Clips accumulate (new clips intersect with existing ones).
  • Use with save()/restore() to limit clip scope.

Common uses: circular image masks, complex shape viewports, masking effects. Clipping is relatively expensive, so use it sparingly.

clipRect

void clipRect(Rect rect, { ClipOp clipOp = ClipOp.intersect, bool doAntiAlias = true })

Purpose: Restricts drawing to a rectangular region.

  • rect ‚Äì Rectangle defining the clip region.
  • clipOp ‚Äì How to combine with existing clips (usually ClipOp.intersect).
  • doAntiAlias ‚Äì Smooth edges (default: true).

Common uses: rectangular viewports, preventing drawing outside bounds, basic optimization.

clipRRect

void clipRRect(RRect rrect, { bool doAntiAlias = true })

Purpose: Restricts drawing to a rounded-rectangle region.

Common uses: rounded card masks, rounded image clipping, pill-shaped viewports.

drawArc

void drawArc(Rect rect, double startAngle, double sweepAngle,
  bool useCenter, Paint paint)

Purpose: Draws an arc (portion of an ellipse or circle).

  • rect ‚Äì Bounding box for the circle/ellipse.
  • startAngle ‚Äì Start angle in radians (0 = 3 o'clock).
  • sweepAngle ‚Äì How far to sweep in radians (positive = clockwise).
  • useCenter ‚Äì true draws a pie slice, false draws just the arc.

Common uses: circular progress indicators, pie charts, gauge dials, Pac‚ÄëMan shapes.

drawCircle

void drawCircle(Offset c, double radius, Paint paint)

Purpose: Draws a perfect circle at the given center with the given radius.

Common uses: dots, indicators, avatars, circular buttons, decorative shapes.

drawLine

void drawLine(Offset p1, Offset p2, Paint paint)

Purpose: Draws a straight line between two points.

Common uses: axes, grid lines, separators, chart lines, connectors.

drawRect and drawRRect

void drawRect(Rect rect, Paint paint)
void drawRRect(RRect rrect, Paint paint)

Purpose: Draws rectangles and rounded rectangles.

  • drawRect ‚Äì plain rectangle (super fast, great for grids, bars, backgrounds).
  • drawRRect ‚Äì rounded rectangle (cards, buttons, modern UI surfaces).

drawPath

void drawPath(Path path, Paint paint)

Purpose: Draws an arbitrary shape defined by a Path.

Common uses: custom polygons, curves, wave shapes, freehand drawing, signatures.

drawPoints / drawRawPoints

void drawPoints(PointMode pointMode, List<Offset> points, Paint paint)
void drawRawPoints(PointMode pointMode, Float32List points, Paint paint)

Purpose: Efficiently draws many points, lines, or polygons in one call.

Common uses: scatter plots, line graphs, particle systems, grids.

drawImage, drawImageRect, drawImageNine

void drawImage(ui.Image image, Offset offset, Paint paint)
void drawImageRect(ui.Image image, Rect src, Rect dst, Paint paint)
void drawImageNine(ui.Image image, Rect center, Rect dst, Paint paint)
  • drawImage ‚Äì draw an image at a position, 1:1 size.
  • drawImageRect ‚Äì draw part of an image, scaled into a destination rect.
  • drawImageNine ‚Äì nine‚Äëpatch rendering for scalable borders/backgrounds.

drawColor and drawPaint

void drawColor(Color color, BlendMode blendMode)
void drawPaint(Paint paint)

Purpose: Fill the entire canvas (or clip region) with a color or paint.

  • drawColor ‚Äì solid color fill using a specific BlendMode.
  • drawPaint ‚Äì fill using a full Paint (gradients, shaders, etc.).

drawShadow

void drawShadow(Path path, Color color, double elevation, bool transparentOccluder)

Purpose: Draws a Material‚Äëstyle drop shadow for the given shape.

üìê Part 3: The Size Object

The Size object is the second parameter in your paint() method. It tells you how much space you have to draw.

@override
void paint(Canvas canvas, Size size) {
  // 'size' tells you how much space you have
}

What is Size?

Size represents 2D dimensions ‚Äì width and height ‚Äì in logical pixels.

class Size {
  final double width;
  final double height;
}

Core properties

  • width ‚Äì horizontal dimension in logical pixels.
  • height ‚Äì vertical dimension in logical pixels.
  • aspectRatio ‚Äì width / height.
  • longestSide ‚Äì max(width, height).
  • shortestSide ‚Äì min(width, height).
  • flipped ‚Äì returns Size(height, width).

Where size comes from

The size parameter is decided by:

  1. The size property of CustomPaint (if you set it).
  2. The child widget's size (if CustomPaint has a child).
  3. The parent constraints (if no explicit size and no child).
CustomPaint(
  size: Size(200, 100), // size will be exactly 200x100
  painter: MyPainter(),
)

Common Size patterns

Center point

final center = Offset(size.width / 2, size.height / 2);

Responsive positioning

final topLeft = Offset(size.width * 0.1, size.height * 0.1);
final bottomRight = Offset(size.width * 0.9, size.height * 0.9);

Creating rects from Size

// Full canvas rect
final rect = Offset.zero & size; // Rect.fromLTWH(0, 0, width, height)

// Centered rect using 80% of the canvas
final centerRect = Rect.fromCenter(
  center: Offset(size.width / 2, size.height / 2),
  width: size.width * 0.8,
  height: size.height * 0.8,
);

Responsive drawing using Size

@override
void paint(Canvas canvas, Size size) {
  final padding = size.width * 0.1; // 10% padding
  final radius = size.shortestSide * 0.4; // 40% of smallest side

  final center = Offset(size.width / 2, size.height / 2);
  canvas.drawCircle(center, radius, Paint()..color = Colors.blue);
}

Handling aspect ratios

@override
void paint(Canvas canvas, Size size) {
  if (size.aspectRatio > 1.5) {
    // Wide layout
    drawWideLayout(canvas, size);
  } else {
    // Narrow or square layout
    drawNarrowLayout(canvas, size);
  }
}
