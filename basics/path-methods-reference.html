üìö Overview

This page provides a comprehensive reference for all methods available in Flutter's Path class. The Path class is essential for creating custom shapes, curves, and complex graphics in CustomPainter.

Path Methods Categories:

  • Path Building: Methods to construct paths (moveTo, lineTo, curves, arcs)
  • Adding Shapes: Methods to add predefined shapes (rectangles, ovals, polygons)
  • Path Operations: Methods to manipulate paths (close, reset, shift, transform)
  • Path Metrics: Methods to measure and analyze paths

üî® Path Building Methods

relativeLineTo(double dx, double dy)

Purpose: Draw a line relative to the current point. Adds dx and dy to the current position instead of using absolute coordinates.

Common uses: Drawing shapes where you know the relative movement, creating patterns, simplifying coordinate calculations.

@override
void paint(Canvas canvas, Size size) {
  final paint = Paint()
    ..color = Colors.blue
    ..style = PaintingStyle.stroke
    ..strokeWidth = 3;

  final path = Path();

  // Starting point
  path.moveTo(50, size.height - 50);

  const stepWidth = 40.0;
  const stepHeight = 30.0;
  const steps = 5;

  for (int i = 0; i < steps; i++) {
    // Move right (horizontal step)
    path.relativeLineTo(stepWidth, 0);

    // Move up (vertical step)
    path.relativeLineTo(0, -stepHeight);
  }

  canvas.drawPath(path, paint);
}

arcToPoint(Offset endRadius, {Radius radius, double rotation, bool largeArc, bool clockwise})

Purpose: Add an arc from the current point to a target point, similar to SVG arc commands.

Parameters:

  • endRadius - Target end point of the arc
  • radius - Radius of the arc (optional)
  • rotation - Rotation of the ellipse in radians (optional)
  • largeArc - If true, use the larger arc; if false, use the smaller arc
  • clockwise - If true, arc goes clockwise; if false, counter-clockwise

Common uses: SVG-like arcs, connecting points with curves, complex path segments.

@override
void paint(Canvas canvas, Size size) {
  final roadPaint = Paint()
    ..color = Colors.grey.withValues(alpha: 0.4)
    ..style = PaintingStyle.stroke
    ..strokeWidth = 10
    ..strokeCap = StrokeCap.round;

  final path = Path();

  // Start of the road (coming from left)
  path.moveTo(50, size.height / 2);

  // Straight road
  path.lineTo(size.width / 2 - 60, size.height / 2);

  // Smooth curved turn (right turn)
  path.arcToPoint(
    Offset(size.width / 2, size.height / 2 + 60),
    radius: const Radius.circular(60),
    clockwise: true,
    largeArc: false,
  );

  // Continue road downward
  path.lineTo(size.width / 2, size.height - 50);

  canvas.drawPath(path, roadPaint);

  // Optional: draw guide points
  canvas.drawCircle(Offset(size.width / 2 - 60, size.height / 2), 5, Paint()..color = Colors.red);
  canvas.drawCircle(Offset(size.width / 2, size.height / 2 + 60), 5, Paint()..color = Colors.blue);
}

Visual guide showing how arcToPoint() creates smooth curved turns (like a road) between two points. The red circle marks the start of the arc, and the blue circle marks the end point.

relativeQuadraticBezierTo(double x1, double y1, double x2, double y2)

Purpose: Draw a quadratic Bezier curve relative to the current point. Similar to quadraticBezierTo(), but uses relative coordinates.

Parameters:

  • x1, y1 - Control point relative to current position
  • x2, y2 - End point relative to current position

Common uses: Creating curves when you know relative movements, building patterns with curves, simplifying coordinate calculations for Bezier curves.

@override
void paint(Canvas canvas, Size size) {
  final wavePaint = Paint()
    ..color = Colors.blue
    ..style = PaintingStyle.stroke
    ..strokeWidth = 4
    ..strokeCap = StrokeCap.round;

  final guidePaint = Paint()
    ..color = Colors.grey.withValues(alpha: 0.5)
    ..style = PaintingStyle.stroke
    ..strokeWidth = 1;

  final path = Path();

  // Start point at the center-left
  final startX = 50.0;
  final startY = size.height / 2;
  path.moveTo(startX, startY);

  // Wave parameters
  const segmentWidth = 30.0; // Width of each wave segment
  const amplitude = 30.0; // Height of the wave peaks/valleys
  const numWaves = 5; // Number of complete wave cycles

  // Draw wave pattern using relativeQuadraticBezierTo
  // Each curve is drawn relative to the current position
  for (int i = 0; i < numWaves; i++) {
    // Curve UP (peak)
    // Control point: halfway horizontally, full amplitude up (relative)
    // End point: full segment width to the right, back to center (relative)
    path.relativeQuadraticBezierTo(
      segmentWidth / 2, // control point x: halfway through segment (relative)
      -amplitude, // control point y: up by amplitude (relative)
      segmentWidth, // end point x: full segment width to the right (relative)
      0, // end point y: back to center line (relative)
    );

    // Curve DOWN (valley)
    // Control point: halfway horizontally, full amplitude down (relative)
    // End point: full segment width to the right, back to center (relative)
    path.relativeQuadraticBezierTo(
      segmentWidth / 2, // control point x: halfway through segment (relative)
      amplitude, // control point y: down by amplitude (relative)
      segmentWidth, // end point x: full segment width to the right (relative)
      0, // end point y: back to center line (relative)
    );
  }

  // Draw the wave path
  canvas.drawPath(path, wavePaint);

  // Draw center line for reference
  canvas.drawLine(Offset(0, size.height / 2), Offset(size.width, size.height / 2), guidePaint);

  // Draw start point marker
  canvas.drawCircle(Offset(startX, startY), 6, Paint()..color = Colors.green);
}

Visual guide showing how relativeQuadraticBezierTo() creates a wave pattern using relative coordinates. The green circle marks the start point, and the gray line serves as a center reference. Each wave segment is created relative to the previous point, demonstrating the power of relative path building.

relativeCubicTo(double x1, double y1, double x2, double y2, double x3, double y3)

Purpose: Draw a cubic Bezier curve relative to the current point. Similar to cubicTo(), but uses relative coordinates.

Parameters:

  • x1, y1 - First control point relative to current position
  • x2, y2 - Second control point relative to current position
  • x3, y3 - End point relative to current position

Common uses: Creating complex curves with relative movements, building wave patterns, simplifying coordinate calculations for cubic Bezier curves.

@override
void paint(Canvas canvas, Size size) {
  final paint = Paint()
    ..color = Colors.blue
    ..style = PaintingStyle.fill;

  final path = Path();

  // Start from top-left
  path.moveTo(0, 0);

  // Go down a bit
  path.lineTo(0, 120);
  final startPoint = Offset(0, 120);

  // Calculate control points relative to startPoint (where relativeCubicTo begins)
  final controlPoint1 = startPoint.translate(size.width * 0.25, -60);
  final controlPoint2 = startPoint.translate(size.width * 0.75, 60);
  final endPoint = startPoint.translate(size.width, 0);

  // Elastic wave using relativeCubicTo
  path.relativeCubicTo(
    size.width * 0.25,
    -60, // control point 1 (relative)
    size.width * 0.75,
    60, // control point 2 (relative)
    size.width,
    0, // end point (relative)
  );

  // Close shape
  path.lineTo(size.width, 0);
  path.close();

  // Draw the path first
  canvas.drawPath(path, paint);

  // Draw control points and end point with visible markers
  final markerPaint = Paint()
    ..color = Colors.red
    ..style = PaintingStyle.fill;

  // Draw control point 1
  canvas.drawCircle(controlPoint1, 8, markerPaint);

  // Draw control point 2
  canvas.drawCircle(controlPoint2, 8, markerPaint);

  // Draw end point
  canvas.drawCircle(endPoint, 8, Paint()..color = Colors.green);
}

Visual guide showing how relativeCubicTo() creates an elastic wave pattern with two control points using relative coordinates. The red circles mark the two control points, and the green circle marks the end point of the curve.

conicTo(double x1, double y1, double x2, double y2, double w)

Purpose: Draw a conic curve (rational quadratic Bezier curve) with a weight parameter. The weight controls how much the curve is pulled towards the control point.

Parameters:

  • x1, y1 - Control point that influences the curve
  • x2, y2 - End point of the curve
  • w - Weight parameter (typically 0.0 to 1.0). Higher values pull the curve more towards the control point

Common uses: Creating curves with adjustable sharpness, drawing arcs and circles precisely, advanced path design with weight control.

//! ConicTo
/// ConicTo is used to draw a conic curve (quadratic Bezier with weight) from the current point
class ConicNotchPainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final strokePaint = Paint()
      ..style = PaintingStyle.stroke
      ..strokeWidth = 3;

    final textPainter = TextPainter(textDirection: TextDirection.ltr);

    // Example 1: Different weights showing curve variations
    final weights = [0.5, 0.707, 1.0, 1.5];
    final colors = [Colors.blue, Colors.green, Colors.orange, Colors.purple];
    final startY = 80.0;
    const spacing = 120.0;

    for (int i = 0; i < weights.length; i++) {
      final path = Path();
      final startX = 50.0;
      final y = startY + (i * spacing);
      final controlX = size.width / 2;
      final endX = size.width - 50;

      path.moveTo(startX, y);
      path.conicTo(
        controlX,
        y - 50, // control point
        endX,
        y, // end point
        weights[i], // weight parameter
      );

      strokePaint.color = colors[i];
      canvas.drawPath(path, strokePaint);

      // Draw control point
      canvas.drawCircle(Offset(controlX, y - 40), 5, Paint()..color = colors[i].withValues(alpha: 0.7));

      // Draw weight label
      textPainter.text = TextSpan(
        text: 'Weight: ${weights[i]}',
        style: TextStyle(color: colors[i], fontSize: 12, fontWeight: FontWeight.bold),
      );
      textPainter.layout();
      textPainter.paint(canvas, Offset(startX, y - 60));
    }

    // Example 2: Perfect circle using conicTo (weight = 0.707)
    final circlePath = Path();
    final centerX = size.width / 2;
    final centerY = size.height - 150;
    final radius = 60.0;

    // Draw circle using 4 conicTo segments
    circlePath.moveTo(centerX, centerY - radius);
    circlePath.conicTo(centerX + radius, centerY - radius, centerX + radius, centerY, 0.707);
    circlePath.conicTo(centerX + radius, centerY + radius, centerX, centerY + radius, 0.707);
    circlePath.conicTo(centerX - radius, centerY + radius, centerX - radius, centerY, 0.707);
    circlePath.conicTo(centerX - radius, centerY - radius, centerX, centerY - radius, 0.707);
    circlePath.close();

    strokePaint.color = Colors.red;
    strokePaint.strokeWidth = 4;
    canvas.drawPath(circlePath, strokePaint);

    // Label for circle
    textPainter.text = const TextSpan(
      text: 'Perfect Circle (weight = 0.707)',
      style: TextStyle(color: Colors.red, fontSize: 14, fontWeight: FontWeight.bold),
    );
    textPainter.layout();
    textPainter.paint(canvas, Offset(centerX - textPainter.width / 2, centerY + radius + 20));
  }
}

Visual guide showing how conicTo() creates curves with different weight parameters (0.5, 0.707, 1.0, 1.5) demonstrating how weight affects curve sharpness. The example also shows how to draw a perfect circle using four conicTo() segments with weight 0.707.

relativeConicTo(double x1, double y1, double x2, double y2, double w)

Purpose: Draw a conic curve relative to the current point. Similar to conicTo(), but uses relative coordinates.

Parameters:

  • x1, y1 - Control point relative to current position
  • x2, y2 - End point relative to current position
  • w - Weight parameter (typically 0.0 to 1.0)

Common uses: Creating weighted curves with relative movements, building patterns with adjustable curve sharpness, simplifying coordinate calculations for conic curves.

@override
void paint(Canvas canvas, Size size) {
  final strokePaint = Paint()
    ..style = PaintingStyle.stroke
    ..strokeWidth = 3;

  final fillPaint = Paint()..style = PaintingStyle.fill;

  // Example: Decorative wave border using relativeConicTo
  final path = Path();
  final startY = size.height / 2;
  final startX = 50.0;

  path.moveTo(startX, startY);

  // Create a wave pattern using relativeConicTo
  // Each curve is drawn relative to the previous position
  const segmentWidth = 70.0;
  const waveHeight = 50.0;
  const numWaves = 2;

  for (int i = 0; i < numWaves; i++) {
    // Upward curve (peak)
    path.relativeConicTo(
      segmentWidth / 2, // control point x (relative)
      -waveHeight, // control point y (relative) - up
      segmentWidth, // end point x (relative)
      0, // end point y (relative) - back to center
      0.707, // weight for smooth circular arc
    );

    // Downward curve (valley)
    path.relativeConicTo(
      segmentWidth / 2, // control point x (relative)
      waveHeight, // control point y (relative) - down
      segmentWidth, // end point x (relative)
      0, // end point y (relative) - back to center
      0.707, // weight for smooth circular arc
    );
  }

  // Close the path to create a filled shape
  path.lineTo(startX + (numWaves * segmentWidth * 2), size.height);
  path.lineTo(startX, size.height);
  path.close();

  // Draw filled wave
  fillPaint.color = Colors.blue.withValues(alpha: 0.3);
  canvas.drawPath(path, fillPaint);

  // Draw stroke
  strokePaint.color = Colors.blue;
  canvas.drawPath(path, strokePaint);

  // Draw start point marker
  canvas.drawCircle(Offset(startX, startY), 6, Paint()..color = Colors.green);
}

Visual guide showing how relativeConicTo() creates a decorative wave pattern using relative coordinates. The green circle marks the start point, and the wave pattern is created with weight 0.707 for smooth circular arcs. The shape is filled with semi-transparent blue and outlined with a solid blue stroke.

üîß Path Operations

close()

Purpose: Close the current contour by drawing a line from the current point back to the starting point.

Common uses: Closing polygons, creating filled shapes, ensuring paths are complete.

@override
void paint(Canvas canvas, Size size) {
  final path = Path();
  final paint = Paint()..color = Colors.blue;

  path.moveTo(50, 50);
  path.lineTo(150, 50);
  path.lineTo(150, 150);
  path.lineTo(50, 150);
  path.close(); // Closes back to (50, 50)

  canvas.drawPath(path, paint);
}

reset()

Purpose: Clear all path data, resetting it to an empty path.

  canvas.drawPath(path, Paint()..color = Colors.blue);
}

üìè Path Metrics

computeMetrics()

Purpose: Get measurements of path segments. Returns a PathMetrics object that provides information about the path's length, contours, and segments.

Common uses: Measuring path length, animating along paths, calculating distances, analyzing path structure.

@override
void paint(Canvas canvas, Size size) {
  final path = Path();
  path.addRect(Rect.fromLTWH(50, 50, 100, 100));

  // Get path metrics
  final metrics = path.computeMetrics();
  final metric = metrics.first;

  // Get path length
  final length = metric.length;

  // Get point at specific distance along path
  final tangent = metric.getTangentForOffset(length / 2);
  if (tangent != null) {
    final point = tangent.position;
    // Use point for drawing or calculations
  }

  canvas.drawPath(path, Paint()..color = Colors.blue);
}

Visual guide showing how computeMetrics() provides path measurements and allows accessing points along the path.

üí° Key Points:
  • computeMetrics() returns an iterable of PathMetric objects
  • Each PathMetric represents one contour in the path
  • Use metric.length to get the total length of the contour
  • Use getTangentForOffset() to get position and direction at a specific distance
  • Useful for path animations and measurements
